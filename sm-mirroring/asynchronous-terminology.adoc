= 了解 SANtricity 軟體中的非同步鏡像術語
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
了解非同步鏡像術語如何套用至您的儲存陣列。

[cols="25h,~"]
|===
| 期限 | 說明 


 a| 
本機儲存陣列
 a| 
本機儲存陣列是您正在操作的儲存陣列。

當您在「本機角色」欄位中看到 *Primary* 時，表示儲存陣列包含在鏡像關係中擔任主要角色的磁碟區。當您在「本機角色」欄位中看到 *Secondary* 時，表示儲存陣列包含在鏡像關係中擔任次要角色的磁碟區。



 a| 
鏡像一致性群組
 a| 
鏡像一致性群組是容納一個或多個鏡像配對的容器。對於非同步鏡像操作，您必須建立鏡像一致性群組。



 a| 
鏡像配對
 a| 
鏡像配對由兩個磁碟區組成，一個主要磁碟區和一個次要磁碟區。

在非同步鏡像中，鏡像配對始終屬於鏡像一致性群組。寫入作業會先在主要磁碟區上執行，然後複寫到次要磁碟區。鏡像一致性群組中的每個鏡像配對都共用相同的同步設定。



 a| 
主要磁碟區
 a| 
鏡像配對的主要磁碟區是要進行鏡像的來源磁碟區。



 a| 
遠端儲存陣列
 a| 
遠端儲存陣列通常被指定為次要站台，通常在鏡射組態中保存資料的複本。



 a| 
預留容量
 a| 
預留容量是指用於任何複製服務作業和儲存物件的實體分配容量。主機無法直接讀取該容量。



 a| 
角色變更
 a| 
角色變更是指將主要角色指派給次要磁碟區，反之亦然。



 a| 
次要磁碟區
 a| 
鏡像對的次級磁碟區通常位於次級站台，並保存資料的複本。



 a| 
同步
 a| 
同步操作發生在本機儲存陣列和遠端儲存陣列之間的初始同步階段。當主磁碟區和輔助磁碟區在通訊中斷後失去同步時，也會進行同步操作。當通訊鏈路恢復正常後，所有未複製的資料都會同步到輔助磁碟區的儲存陣列。

|===
