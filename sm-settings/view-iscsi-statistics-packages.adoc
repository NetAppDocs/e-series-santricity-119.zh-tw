= 在 SANtricity System Manager 中檢視 iSCSI 統計資料套件
:allow-uri-read: 
:experimental: 
:icons: font
:imagesdir: ../media/


[role="lead"]
您可以查看有關儲存陣列的 iSCSI 連線的資料。

.關於此任務
System Manager 會顯示這些類型的 iSCSI 統計資料。所有統計資訊均為唯讀，無法設定。

* *乙太網路 MAC 統計資料* -- 提供媒體存取控制（MAC）的統計資料。MAC 也提供了一種尋址機制，稱為實體位址或 MAC 位址。MAC 位址是分配給每個網路介面卡的唯一位址。MAC 位址有助於將資料封包傳遞到子網路內的目的地。
* *乙太網路 TCP/IP 統計資訊* -- 提供 iSCSI 設備的 TCP/IP 統計訊息,TCP/IP 是傳輸控制協定 (TCP) 和網際網路協定 (IP) 的結合體。透過 TCP,聯網主機上的應用程式可以相互建立連接,並透過這些連接以封包的形式交換資料。IP 是一種以資料為導向的協定,用於在封包交換的網際網路上傳輸資料。IPv4 統計資訊和 IPv6 統計資訊分別顯示。
* *本地目標 / 啟動器（通訊協定）統計資料* -- 顯示 iSCSI 目標的統計資料，該目標提供對其儲存媒體的區塊層級存取，並顯示儲存陣列在非同步鏡射作業中作為啟動器使用時的 iSCSI 統計資料。
* *DCBX 運作狀態統計資料* -- 顯示各種 Data Center Bridging Exchange (DCBX) 功能的運作狀態。
* *LLDP TLV 統計資料* -- 顯示連結層發現協定（LLDP）類型長度值（TLV）統計資料。
* *DCBX TLV 統計資料* -- 顯示在 Data Center Bridging (DCB) 環境中識別儲存陣列主機連接埠的資訊。此資訊會與網路對等方共用，以供識別和功能用途。


您可以將每項統計資料檢視為原始統計資料或基準統計資料。原始統計資料是指自控制器啟動以來收集的所有統計資料。基準統計資料是指自您設定基準時間以來收集的特定時間點統計資料。

.步驟
. 選擇功能表：Settings[System]。
. 選擇 *View iSCSI Statistics Packages*。
. 點擊標籤頁即可檢視不同的統計資料集。
. *選用：* 要設定基線，請點選 *Set new baseline*。
+
設定基線為統計資訊的收集設定了一個新的起點。所有 iSCSI 統計資訊都使用相同的基線。



如需有關不同類型統計資料的詳細資訊，請參閱以下章節。

== DCBX 運作狀態統計資料

選擇 Data Center Bridging Exchange （DCBX） 運作狀態統計資料時，將顯示 DCBX 統計資料。

.DCBX 統計數據：
[%collapsible]
====
[cols="25h,~"]
|===
| 統計 | 定義 


 a| 
iSCSI 主機連接埠
 a| 
以「控制器 #、連接埠 #」格式指示偵測到的主機連接埠的位置。



 a| 
優先群組
 a| 
指示優先群組（PG）應用程式的運作狀態。狀態為「已啟用」或「已停用」。



 a| 
基於優先順序的流量控制
 a| 
指示基於優先權的流量控制（PFC）功能的運作狀態。狀態為「已啟用」或「已停用」。



 a| 
iSCSI 功能
 a| 
指示網際網路小型電腦系統介面（iSCSI）應用程式的運作狀態。狀態為「已啟用」或「已停用」。



 a| 
FCoE 頻寬
 a| 
指示乙太網路光纖通道（FCoE）頻寬的狀態。狀態為 True 或 False。



 a| 
無 FCoE/FIP 對應不符
 a| 
指示 FCoE 和 FCoE 初始化協定（FIP）之間是否存在映射不符。該值為 True 或 False。

|===
====

NOTE: 您可以在狀態擷取檔案中找到其他 DCBX 運作狀態統計資料。



== LLDP TLV 統計資料

選擇 Link Layer Discovery Protocol (LLDP) Type Length Value (TLV) Statistics 時，將顯示兩組統計資料：一組用於本機設備，一組用於遠端設備。本機設備指的是控制器。遠端設備指的是與控制器連接的對等設備，通常是交換器。

.LLDP TLV 統計資料：
[%collapsible]
====
[cols="25h,~"]
|===
| 統計 | 定義 


 a| 
iSCSI 主機連接埠
 a| 
以「控制器 #、連接埠 #」格式指示偵測到的主機連接埠的位置。



 a| 
機箱 ID
 a| 
表示機箱 ID。



 a| 
機箱 ID 子類型
 a| 
表示機箱 ID 子類型。



 a| 
連接埠 ID
 a| 
指示連接埠 ID。



 a| 
連接埠 ID 子類型
 a| 
指示連接埠 ID 子類型。



 a| 
存留時間
 a| 
表示接收方 LLDP 代理程式認為資訊有效的秒數。

|===
====

NOTE: 您可以在狀態擷取檔案中找到其他 LLDP TLV 統計資料。



== DCBX TLV 統計資料

選擇 Data Center Bridging Exchange (DCBX) Type Length Value (TLV) Statistics 時，將顯示下列統計資料：

* *本地統計資料* -- 出廠時在控制器上設定的 DCBX 參數。
* *運作統計資料* -- DCBX 協商產生的 DCBX 參數。
* *遠端統計資訊* -- 控制器所連接的對等裝置（通常是交換器）的 DCBX 參數。


.DCBX TLV 統計資料：
[%collapsible]
====
[cols="25h,~"]
|===
| 統計 | 定義 


 a| 
iSCSI 主機連接埠
 a| 
以「控制器 #、連接埠 #」格式指示偵測到的主機連接埠的位置。



 a| 
流量控制模式
 a| 
整個連接埠的流量控制模式。有效值為 Disabled、Standard、Per Priority 或 Indeterminate。



 a| 
傳輸協定
 a| 
通訊協定。有效值為 FCoE 、 FIP 、 iSCSI 或 UNKNOWN 。



 a| 
優先順序
 a| 
表示通訊優先順序編號的整數值。



 a| 
優先群組
 a| 
表示已指派給協定之優先順序群組的整數值。



 a| 
優先群組 % 頻寬
 a| 
百分比值，表示分配給優先權群組的頻寬量。



 a| 
DCBX PFC 狀態
 a| 
特定連接埠的基於優先權的流量控制（PFC）狀態。該值為啟用或停用。

|===
====

NOTE: 您可以在狀態擷取檔案中找到其他 DCBX TLV 統計資料。
